# ReUse – Plataforma de Trocas Sustentáveis

## Contexto da Sprint

Esta é a primeira fase do projeto **ReUse**, uma plataforma digital voltada à economia sustentável, reutilização de produtos e consumo consciente.

A proposta da aplicação é permitir que usuários publiquem objetos que desejam trocar, criando uma rede colaborativa de troca de bens sem a necessidade de transações financeiras. Dessa forma, o projeto busca incentivar práticas de reaproveitamento, reduzir o descarte de objetos ainda utilizáveis e promover um modelo de consumo mais consciente.

Nesta **SPRINT inicial**, o foco do desenvolvimento está na construção do **primeiro contato do usuário com o aplicativo**, priorizando:

- uma interface mobile simples e intuitiva;
- organização clara das telas;
- navegação básica entre as principais funcionalidades;
- estrutura inicial da aplicação para futuras expansões.

O objetivo principal desta fase é validar o conceito da plataforma e estabelecer a base da experiência do usuário dentro do aplicativo.

---

## Decisões de Interface e Organização do App

O aplicativo foi desenvolvido utilizando **React Native**, framework que permite criar aplicações mobile utilizando JavaScript e componentes reutilizáveis, facilitando o desenvolvimento multiplataforma para Android e iOS.

Durante o desenvolvimento desta sprint, algumas decisões de design e organização foram adotadas para garantir clareza e facilidade de uso.

### Interface

A interface foi projetada com foco em:

- **Simplicidade visual**
- **Navegação clara**
- **Facilidade de compreensão para novos usuários**

As principais diretrizes adotadas foram:

- Layout limpo, com poucos elementos por tela
- Hierarquia visual clara
- Uso de componentes reutilizáveis
- Estrutura preparada para futuras funcionalidades da plataforma

O objetivo é permitir que o usuário compreenda rapidamente a proposta do aplicativo e consiga navegar entre as telas sem dificuldade.

### Organização do Aplicativo

A aplicação segue uma estrutura modular, comum em projetos React Native, separando responsabilidades entre telas, componentes e arquivos de configuração.

As principais decisões de organização incluem:

- Separação entre telas da aplicação
- Componentização de elementos reutilizáveis
- Estrutura de pastas organizada para facilitar manutenção
- Código preparado para escalabilidade em futuras sprints

Essa organização facilita a evolução do projeto, permitindo adicionar novas funcionalidades como cadastro de produtos, sistema de trocas e comunicação entre usuários.

## Estrutura do Projeto

A estrutura do projeto foi organizada para manter o código limpo, modular e de fácil manutenção.


### Descrição dos principais diretórios

**assets**

Contém imagens, ícones e demais recursos visuais utilizados na interface do aplicativo.

**components**

Armazena componentes reutilizáveis utilizados em diferentes partes da aplicação, permitindo maior organização e reaproveitamento de código.

**screens**

Contém as telas principais do aplicativo, responsáveis por estruturar a navegação e apresentar as funcionalidades para o usuário.

**App.js**

Arquivo principal da aplicação React Native, responsável por iniciar o aplicativo e organizar a estrutura geral de navegação.

**package.json**

Arquivo que define as dependências do projeto, scripts de execução e informações gerais da aplicação.

---

## Objetivo do Projeto

O projeto **ReUse** busca promover um modelo de consumo mais sustentável, incentivando a reutilização de objetos e a troca entre usuários.

Ao invés de comprar novos produtos ou descartar itens ainda utilizáveis, os usuários podem oferecer seus objetos em troca de outros, criando um ecossistema colaborativo baseado na economia circular.
